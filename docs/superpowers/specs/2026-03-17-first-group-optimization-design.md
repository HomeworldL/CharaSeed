# CharaSeed 第一组优化设计

日期：2026-03-17

## 目标

这一轮只处理第一组优化，不混入站点大扩容。

目标分为五类：

1. 搜索页改成单站点查询，速度更稳定，站点语境更明确。
2. 主页保留混合候选流，但改成分站点渐进补齐，不再等待整批结果。
3. 今日候选流加入更明显的随机扰动和近期曝光惩罚，避免刷新几乎不变。
4. 收藏链路补齐自动填充、删除、中文状态和手动补录整合。
5. 预览和列表视觉修正，包括更清晰的卡片预览图、原图预览和卡片动作区对齐。

## 交互调整

### 搜索页

- 只允许先选一个站点再搜索，不再提供 `All` 聚合搜索。
- 搜索框位置保持不变。
- 站点切换继续使用按钮式 segmented control。
- 不再暴露 `image / figure` 这类二级限制。
- 搜索结果卡片根据站点展示不同细节字段。

### 主页

- 主页继续是混合候选流，不和搜索页混用。
- 刷新主页时先返回首页框架，再由各站点分批补齐卡片。
- 点击站点按钮后，在同页下方展开该站点专区，并支持单站刷新。
- 移除顶部冗余解释块，只保留简洁的主页操作区。

### 收藏状态

- 第一版状态简化为：
  - `archived`：已归档
  - `pending`：待整理
- 点击收藏默认进入 `已归档`。
- 旧状态值在读取时兼容映射：
  - `collected -> archived`
  - `wishlist -> pending`

## 数据与服务策略

### 搜索性能

- 搜索页每次只查询一个站点。
- 搜索服务增加短时内存缓存，减少重复请求。
- 站点请求尊重系统代理环境，不再强制 `trust_env=False`。
- 缩短单站超时，慢站直接降级，不阻塞其他主页站点分片。

### 首页渐进加载

- `/fragments/home-feed` 只负责返回主页候选流外壳。
- 外壳内按站点创建异步 fragment loader。
- 每个 loader 独立请求自己的候选卡片并追加到混合瀑布流。
- 首页刷新不再等待“凑满固定数量”。

### 推荐扰动

- 候选查询词仍由收藏历史和搜索历史生成。
- 每次刷新按随机种子重新抽样查询词。
- 引入近期曝光惩罚，降低刚出现过的条目再次被选中的概率。
- 最终排序仍保留兴趣锚点：高频角色、作品、标签、最近搜索、偏好站点。

## 收藏整理能力

- 收藏弹层自动填充角色、作品、作者、厂商等字段。
- 在收藏库和详情页补删除操作。
- 手动补录入口移到收藏库页，不再保留独立新增页面。
- 收藏库筛选改成真正生效，状态筛选只保留 `全部 / 已归档 / 待整理`。

## 视觉修正

- 卡片优先使用更清晰的预览图变体。
- 预览弹层对图片类结果直接展示原图。
- 卡片布局改成固定媒体区 + 可伸缩正文区 + 底部动作区，保证按钮水平线对齐。

## 验证

- 路由 smoke：`/home`、`/discover`、`/library`
- 首页 fragment：主页外壳、单站卡片、单站展开面板
- 收藏创建、编辑、删除
- 收藏库筛选与中文状态展示
- 基础测试与 `compileall`
